# Rossmann 매장 매출 예측 (시계열 ML)

과거 매출·프로모션·휴일 데이터를 활용해 매장의 일별 매출을 예측하는 시계열 머신러닝 모델입니다.
데이터 정제 → 시계열 피처 엔지니어링 → 모델 학습 → 검증의 전 과정을 직접 수행했습니다.

**핵심 성능: Test MAPE 9.62%**

---

| 항목 | 내용 |

| 주제 | 매장 일별 매출 예측 (시계열 회귀) |

## 분석 과정

### 1. 데이터 정제

- 영업일만 남기고 휴무일(`Open=0`)을 제거한 뒤, 영업했는데 매출이 0인 이상치도 제외했습니다. (101만 → 84만 행)
- 숫자와 문자가 섞여 있던 `StateHoliday` 컬럼 타입을 문자열로 통일하고, `Date`를 datetime으로 변환해 시계열 처리의 전제를 맞췄습니다.
- 결측치는 변수 성격에 맞게 처리했습니다.
  - 경쟁점 거리(`CompetitionDistance`)는 극단값 영향을 줄이기 위해 평균이 아닌 중앙값으로 대체
  - 프로모션 미진행 매장의 값은 '결측'이 아니라 '해당 없음'이므로 0 또는 'None'으로 처리
  - 결측이 과도하게 많은 경쟁점 개점 시기 변수는 이번 모델에서 제외

### 2. 시계열 피처 엔지니어링 (핵심 단계)

- **시간 파생 변수**: 연·월·일, 연중 주차(WeekOfYear), 월초·월말 여부
- **시차·이동평균 변수**: 매장별 1일 전·7일 전 매출, 최근 7일 이동평균
- **데이터 누수(Data Leakage) 방지**: 이동평균을 만들 때 `shift(1)`을 먼저 적용해, 예측 시점에 실제로 알 수 있는 과거 정보만 쓰도록 설계했습니다. 오늘 매출을 예측하는 변수에 오늘 매출이 새어 들어가지 않게 한 것이 시계열 설계의 핵심입니다.

### 3. 학습 / 검증 분할

- 랜덤 분할(`train_test_split`)을 쓰지 않고, 가장 최근 6주를 검증셋으로 떼는 **시간 기준 분할**을 사용했습니다.
- 시계열을 무작위로 섞으면 미래 데이터로 학습해 과거를 예측하는 모순(시간 누수)이 생기기 때문입니다.

### 4. 모델 학습

- **LightGBM Regressor** 사용 — 정형 데이터에서 성능이 좋고, 범주형·결측치 처리에 강하며 학습이 빠릅니다.
- `early_stopping(50)`으로 과적합을 방지했습니다.

## 분석 결과

| 지표 | 값 | 의미 |
| --- | --- | --- |
| MAPE | 9.62% | 평균 오차율 — 매출 대비 약 10% 이내 예측 |
| MAE | 648.77 | 평균 절대 오차 |
| RMSE | 915.25 | 큰 오차에 민감한 지표 |

- 평균 매출이 약 6,955인 점을 고려하면 첫 모델로서 MAPE 9.62%는 양호한 수준입니다.
- RMSE(915)가 MAE(648)보다 큰 것은 일부 매장·시점에서 큰 오차가 발생함을 의미하며, 추가 개선 여지로 해석할 수 있습니다.

### 피처 중요도

- `CompetitionDistance`와 `Store`가 가장 중요하게 나타났습니다. → 매출 수준은 시점보다 매장 고유 특성(입지·규모·경쟁 강도)에 더 크게 좌우됩니다.
- 과거 매출(`Sales_lag_1`, `Sales_roll_mean_7`, `Sales_lag_7`)도 중상위권에 위치해, 시계열 변수가 예측에 기여함을 확인했습니다.
- 반면 월초·월말 변수(`IsMonthStart/End`)는 중요도가 낮아, 이 데이터에서는 월초·월말 효과가 약하다는 점을 확인했습니다.

## 한계 및 개선 방향

1. **성능 개선 여지** — 1,000번째 트리까지도 검증 성능이 계속 개선되고 있어, 트리 수 증가·하이퍼파라미터 튜닝으로 추가 향상이 기대됩니다.
2. **변수 영향 방향 미해석** — feature importance는 기여도만 보여주므로, SHAP을 활용한 변수별 영향 방향 분석이 필요합니다.
3. **미사용 변수** — 결측이 많아 제외한 경쟁점 개점 시기 변수를 살리면 정보를 추가로 반영할 수 있습니다.
